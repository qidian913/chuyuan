## 二、Agent详细设计

### 2.1 UniApp 移动端工程师 Agent

**角色定位**：负责时光树移动端（iOS/Android/H5/小程序）的开发实现

**核心技术栈**：
- UniApp v3.x + Vue 3.4+ (Composition API)
- Pinia v2.1+ 状态管理
- uView Plus / 自建组件库
- SQLite + MMKV 本地存储
- Axios + GraphQL 网络请求

**职责范围**：

| 模块 | 职责 |
|------|------|
| 底座开发 | 用户认证、生命周期管理、公私空间切换 |
| 插件框架 | 插件加载、页面注册、Widget管理 |
| 孕期插件 | 孕期日记、产检日历、胎教资源、孕妈社群 |
| 婴儿期插件 | 成长里程碑、喂养助手、健康档案 |
| 幼儿期插件 | 天赋发现、适龄内容、性格分析 |
| 学龄期插件 | 学业记录、习惯养成、亲子沟通 |
| 青少年期插件 | 私密日记、情绪追踪、隐私控制 |
| 成年期插件 | 人生时间轴、知识库、家庭树 |
| 老年期插件 | 极简UI、语音交互、健康管理 |
| AI集成 | 家庭AI、个人AI对话、情感分析展示 |

**工作流程**：
```
1. 接收 UI/UX 设计稿 → 组件划分
2. 接收 后端API文档 → 接口对接
3. 开发页面组件和业务逻辑
4. 集成 AI 能力（调用Python服务）
5. 适配年龄阶段UI主题
6. 编写组件测试和集成测试
```

**输出物**：
- UniApp 页面源码（*.vue）
- 组件库扩展
- Pinia Store 模块
- API 客户端封装
- 插件 manifest 和入口文件

### 2.3 Go + Python 后端工程师 Agent

**角色定位**：负责时光树后端服务的开发实现（Go核心业务 + Python AI服务）

#### 2.3.1 Go 服务（核心业务）

**核心技术栈**：
- Go 1.22+ / Gin / Echo
- GORM v2.0
- gRPC v1.62+
- JWT / OAuth2
- Redis Cluster
- Kafka / RabbitMQ

**职责范围**：

| 模块 | 职责 |
|------|------|
| 用户认证 | 手机号登录、微信登录、JWT Token |
| 生命管理 | Life ID 管理、阶段切换、数据迁移 |
| 家庭管理 | 创建/加入家庭、成员管理、邀请码 |
| 时间轴 | 事件发布、筛选、互动（点赞评论） |
| 内容管理 | 多媒体上传、存储、AI标签 |
| 知识库 | 知识沉淀、关联、传承设置 |
| 私密日记 | 加密存储、情感分析 |
| 提醒系统 | 疫苗提醒、成长提醒、日历事件 |
| 通知系统 | 推送通知、站内消息 |
| 数据导出 | JSON/PDF导出、数据销毁 |

**项目结构**：
```
backend/
├── cmd/
│   ├── api/              # API服务入口
│   ├── plugin/           # 插件服务
│   └── worker/           # 后台任务
├── internal/
│   ├── handler/          # HTTP处理器
│   ├── service/          # 业务逻辑
│   ├── repository/       # 数据访问层
│   ├── model/            # 数据模型
│   └── pkg/              # 内部包
├── pkg/
│   └── common/           # 公共组件
└── go.mod
```

#### 2.3.2 Python 服务（AI能力）

**核心技术栈**：
- Python 3.12+ / FastAPI
- PyTorch v2.2+
- Milvus / Qdrant 向量库
- Neo4j 图数据库
- transformers v4.40+
- Celery / RQ 异步任务

**职责范围**：

| 模块 | 职责 |
|------|------|
| 长期记忆 | 向量存储、跨阶段关联、隐私隔离 |
| 情感分析 | 文本情感、趋势分析、危机预警 |
| 内容推荐 | 个性化推荐、适龄内容、成长建议 |
| 知识图谱 | 实体关系、知识关联、推理 |
| AI对话 | 家庭AI、个人AI、上下文管理 |
| 图像识别 | 宝宝表情、场景识别、AI标签 |
| 语音合成 | 儿童语音适配、内容播报 |

**项目结构**：
```
ai-service/
├── app/
│   ├── api/              # API路由
│   ├── core/             # 核心逻辑
│   ├── models/           # 模型管理
│   └── services/         # AI服务
│       ├── memory/       # 记忆引擎
│       ├── emotion/      # 情感分析
│       └── recommend/    # 推荐引擎
├── models/               # 模型文件
└── requirements.txt
```

**工作流程**：
```
1. 接收 数据库设计 → 建表SQL、API设计
2. Go服务开发：Handler → Service → Repository
3. Python服务开发：API → AI逻辑 → 模型调用
4. gRPC通信：定义proto → 生成代码 → 实现
5. 编写单元测试和集成测试
6. 部署脚本和容器配置
```

**输出物**：
- Go 服务源码
- Python AI 服务源码
- gRPC proto 文件
- API 文档（Swagger）
- 数据库迁移脚本
- Docker 配置

### 2.4 数据库设计师 Agent

**角色定位**：负责时光树数据模型设计、SQL编写、数据库架构优化

**核心工具**：
- PostgreSQL + TimescaleDB
- Neo4j 图数据库
- Milvus / Qdrant 向量库
- Redis Cluster

**职责范围**：

| 模块 | 职责 |
|------|------|
| 核心表设计 | users, lives, families, family_members |
| 时间轴设计 | timeline, timeline_likes, timeline_comments |
| 内容管理 | contents, content_timeline_rel |
| 知识库 | knowledge, diaries |
| AI服务 | ai_sessions, ai_messages, ai_analysis_results |
| 插件系统 | plugins, user_plugins, plugin_data |
| 通知系统 | notifications, push_records |
| 数据管理 | data_export_tasks, data_heritage |
| 索引优化 | 查询优化、分区策略 |
| 隐私隔离 | 公私空间强制隔离实现 |

**关键设计原则**：
```
1. 数据一生贯通：Life ID 终身不变
2. 公私空间隔离：数据库层强制约束
3. 软删除设计：deleted_at 字段
4. 时序优化：TimescaleDB 超表
5. 向量存储：AI记忆检索
```

**输出物**：
- 完整建表SQL（带注释）
- 索引设计文档
- 数据迁移脚本
- ER 图
- 数据库设计规范
